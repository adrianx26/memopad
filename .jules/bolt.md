## 2024-03-07 - Pure String Operations for Path Ignore Matching
**Learning:** Instantiating `pathlib.Path` objects and calling `.relative_to()` inside recursive directory traversal loops (like `rglob` or custom `scan_directory`) causes extreme overhead due to object creation. A pure string-based approach is significantly faster but requires careful boundary handling to avoid partial prefix matches (e.g., base `/app/dir` falsely matching `/app/dir2`). Also, replacing a cached pattern matching system with per-call object instantiation in a core utility function (`should_ignore_path`) causes massive regressions if that utility is used in unrefactored loop contexts (`filter_files`).
**Action:** When optimizing loop performance, prefer string slicing with `os.sep` checks. Always ensure utility functions that were previously stateless either retain their caching behavior or verify that *all* call-sites instantiate any new matcher objects *outside* of the loop.
